# Product Spec

本文件是当前项目阶段范围、目标与 in/out of scope 的唯一权威来源。

## 1. 当前阶段

本项目当前处于「首页互动式领养浏览」阶段，目标是在稳定首页结构上补齐前端 mock 的领养互动流程。

## 2. 当前目标

本阶段要完成的内容：

- 首页视觉设计
- 品种分类浏览
- 狗狗资料卡片展示
- 前端 mock 的待认养与确认领养互动
- 前端 session 内单次最多认领 3 只的限制逻辑
- 已认领狗狗清单查看入口
- 领养成功反馈弹窗
- 响应式排版

## 3. 当前范围

本阶段允许：

- 基于既有 React + Vite + TypeScript 项目结构继续开发
- 以前端静态 mock 数据为基础完成首页展示
- 直接基于 `dogs.json` 做品种浏览与卡片渲染
- 在前端 session 内维护待认养、暂时锁定与已认养状态
- 以前端 session 规则限制单次最多认领 3 只，并以已认领数量扣减待认养可加入名额
- 以菜单 / 购物车式交互完成底部待认养结算体验
- 在首页分类浏览区提供已认领狗狗的查看入口
- 维持 Material UI 为当前首页实现方案

## 4. 当前不做的内容

本阶段禁止：

- 后端串接
- 持久化状态
- 真实订单、付款或结算系统
- 复杂筛选、排序、分页
- 登录状态
- 收藏功能
- 弹窗详情流程

## 5. 数据前提

当前阶段以前端 mock 数据为准：

- 数据产物目录：`src/public/data/`
- 数据文件：`dogs.json` 与 `dogs.csv`

字段契约、允许值与同步更新点以 `docs/data-model.md` 为准，本文件不重复列字段表。

## 6. 与实现的关系

当前首页实现应满足以下原则：

- 先让使用者看懂救援犬资料，再自然进入前端 mock 的领养互动
- 首页只承接轻量领养流程，不延伸到真实业务提交
- 状态流与组件边界以 `docs/architecture.md` 和 `docs/homepage-component-contracts.md` 为准

## 7. 变更规则

若要改变当前阶段、目标、允许范围或禁止事项，必须先更新本文件，再同步：

- `docs/guide.md`
- `docs/change-workflow.md`（若流程判断受影响）
- `README.md`（仅摘要层需要变化时）
- `AGENTS.md`（仅门禁引用关系需要变化时）
