# 变更流程

本项目协作一律依照以下顺序进行：

1. 从 `docs/guide.md` 确认本次改动涉及的文档入口。
2. 若改动影响阶段范围或禁止事项，先更新 `docs/spec.md`。
3. 若改动影响资料目录、字段、允许值或同步方式，先更新 `docs/data-model.md`。
4. 若改动影响维护约定、注释策略或协作规则，先更新 `docs/code-maintenance.md`、`README.md` 与 `AGENTS.md`。
5. 文档更新完成后，再进入代码实现。
6. 实作完成后，运行对应脚本或构建命令验证结果，并确认文档与实现一致。

## 注释变更补充

- 新增功能、重构既有逻辑或调整脚本行为时，要一并评估是否需要补充或更新注释。
- 注释应解释「为什么这样做」或「哪里容易误踩」，而不是逐行复述程式本身。
