# 文档导览

本项目协作与实现一律先从本文进入，再依需求跳转到对应专题文件。

## 阅读顺序

1. `docs/change-workflow.md`：确认本次改动应遵守的变更顺序。
2. `docs/spec.md`：确认当前阶段允许开发的范围与门禁。
3. `docs/data-model.md`：确认资料目录、字段契约与允许值。
4. `docs/code-maintenance.md`：确认注释、命名与维护性约定。
5. `README.md`：查看项目摘要、快速启动与对外说明。

## 文档同步原则

- 任何代码改动前，先更新受影响的文档。
- 影响协作规则时，同时更新 `AGENTS.md` 与对应 `docs/` 文件。
- 若摘要文档与专题文档冲突，以专题文档为准，并补齐 `README.md` 摘要。
